mini_sns 라는 프로젝트 디렉토리를 만들어서 아래의 내용대로 구현해줘, supabase mcp를 사용해서 데이터베이스를 구현해서 연결하고, 최종적으로 구현한 후에는 빌드하고 GitHub Pages로 배포까지 todo 계획을 세워서 순차적으로 모두 진행해줘. GitHub Actions 워크플로우를 사용해서 자동 배포되도록 설정하고, 배포 완료 후 접속 가능한 URL을 안내해줘.

모바일 서비스 UI 기획서
1. 서비스 기본 정보
서비스 이름:
POP SPOT (가칭)

서비스 주제:
사설 전시 및 팝업스토어 정보 큐레이션 플랫폼

핵심 가치:
흩어져 있는 사설 전시와 팝업 정보를 지역/위치 기반으로 통합하여 제공하고,
마감 임박 정보와 세부 카테고리 큐레이션을 통해 사용자의 방문 결정을 빠르게 유도하는 플랫폼.
각 팝업/전시를 하나의 “에디토리얼 매거진 콘텐츠”처럼 구성하여 정보와 감성을 함께 전달하는 UX 제공

타겟 사용자:
- 20~30대 팝업/전시/브랜드 공간 관심 사용자
- SNS 기반으로 트렌드를 탐색하는 사용자
- 데이트/나들이/문화 경험을 찾는 사용자

== POP SPOT DB 구조 정리 (아이디 기반) ==
사용자 정보 (users 테이블)
- SNS 로그인/프로필 화면에서 필요한 정보들:
  * user_id: (PK, 자동 증가)
  * username: (@처럼 사용하는 고유 아이디)
  * email: (로그인 및 계정 식별)
  * password: (일반 로그인 시, 소셜 로그인은 NULL 가능)
  * display_name: (프로필에 표시되는 이름 - 선택값)
  * bio: (자기소개)
  * profile_image: (프로필 이미지 URL)
  * created_at: (가입일)

팝업/전시 정보 (posts 테이블)
- 서비스 핵심 콘텐츠
  * post_id: (PK)
  * title: (팝업/전시 이름)
  * description: (에디토리얼 소개)
  * category: (아이돌 / 뷰티 / 패션 / 전시 / 라이프스타일)
  * region: (지역)
  * address: (상세 주소)
  * latitude / longitude: (위치 기반 기능)
  * start_date: (시작일)
  * end_date: (종료일 - D-day 기준)
  * images: (이미지 URL 최대 3개)
  * created_at: (등록일)

좋아요 정보 (likes 테이블)
  * like_id: (PK)
  * user_id: (users 연결)
  * post_id: (posts 연결)
  * created_at: (생성일)

저장 정보 (saves 테이블)
  * save_id: (PK)
  * user_id: (users 연결)
  * post_id: (posts 연결)
  * created_at: (저장일)

리뷰 정보 (reviews 테이블)
  * review_id: (PK)
  * user_id: (users 연결)
  * post_id: (posts 연결)
  * rating: (아주좋음 / 좋음 / 보통 / 나쁨 / 아주나쁨)
  * content: (텍스트 후기)
  * images: (최대 3장)
  * created_at: (작성일)

테이블 관계 구조
- users 1 : N posts
- users 1 : N likes
- users 1 : N saves
- users 1 : N reviews

- posts 1 : N likes
- posts 1 : N saves
- posts 1 : N reviews
