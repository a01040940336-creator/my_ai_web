'my-community'라는 프로젝트를 생성해줘.

작업은 Todo 계획을 세워서 순차적으로 진행해줘.

요구사항:
1. React + Vite로 프로젝트 생성
2. Supabase MCP를 사용해서 데이터베이스 테이블 생성 및 연결
3. 백엔드 없이 Supabase를 직접 연결하여 작동하는 구조
4. GitHub Pages로 배포 (GitHub Actions 워크플로우 사용)

개발 순서:
1) 프로젝트 초기 설정
2) Supabase 데이터베이스 설계 및 테이블 생성
3) 프론트엔드 개발 (기획안 참고)
4) Supabase 연동 구현
5) npm run build로 로컬 빌드
6) GitHub Actions 워크플로우 설정 (.github/workflows/deploy.yml 생성)
7) GitHub에 커밋 및 푸시하여 자동 배포
8) Actions 탭에서 배포 완료 확인 후 접속 가능한 URL 안내

특별한 기능 아이디어
- 기본 기능 외에 추가하고 싶은 것들: (예: 좋아요, 별점, 사진 업로드 등)
- 1. 휴대폰 본인인증 : 회원가입 시 휴대폰 인증 필수,가짜 계정 및 사기 방지
2. 동네 인증 시스템 : 거주 지역 인증,같은 지역 사용자끼리 정보 공유 가능
3. 냉장고 구조대 : 남는 식재료 나눔 게시판, 유통기한 임박 식재료 공유
4. 공동구매 기능: 생수, 휴지, 세제 등 공동구매 모집, 배달비 나눔 기능 제공
5. 생활 도우미 매칭 : 공구 대여, 간단한 생활 도움 요청, 이사 및 청소 정보 공유

페이지별 특징
1. 로그인 페이지
- 로고 및 슬로건 표시
- 아이디 / 비밀번호 입력
- 회원가입 버튼
- 카카오 로그인
- 네이버 로그인

1-2 회원가입 페이지 기능
- 이메일/아이디 중복 확인 기능 제공
- 비밀번호 설정 시 보안 규칙 적용 (8자 이상, 영문/숫자/특수문자 포함)
- 비밀번호 확인 기능을 통해 입력 오류 방지
- 카카오, 네이버 간편 회원가입 지원
- 휴대폰 본인인증 기능을 통해 계정 신뢰도 확보 및 중복/허위 계정 방지
- 닉네임 설정 및 닉네임 중복 확인 기능 제공
- 관심 지역 선택 기능 (동네 기반 콘텐츠 추천을 위한 초기 설정)
- 이용 약관 및 개인정보 처리방침 동의 필수 체크
- 프로필 기본 설정 기능 (기본 아바타 또는 이미지 선택 가능)

2. 메인 홈 페이지
- 추천 게시물 피드
- 인기 게시물 TOP
- 내 지역 게시물 우선 노출
- 카테고리 바로가기 : 자취꿀팁,요리,공동구매,냉장고 나눔,동네정보
- 검색 기능 (게시물/유저)

3. 게시물 목록
- 카테고리 기반 게시글 탐색 기능 제공 -->(자취 꿀팁 / 요리 / 공동구매 / 냉장고 나눔 / 동네 모임 / 질문 게시판)
- 동네 기반 필터 기능 제공 -->  “우리 동네 / 근처 동네 / 전체”로 게시물 범위 선택 가능
- 행동 중심 콘텐츠 분류 --> (냉장고 나눔 / 공동구매 / 생활 도움 / 일반 게시글)
- 카드형 피드 UI 구성 -->이미지, 제목, 동네 정보, 작성 시간, 참여 상태를 포함한 카드 형태
- 참여 유도 기능 -->단순 좋아요/댓글 외에 “참여하기 / 신청하기 / 나눔 요청” 버튼 제공
- 실시간성 반영 --> 인기 게시물 및 최신 게시물 우선 정렬 옵션 제공
- 검색 기능 --> 게시물, 사용자, 키워드 기반 통합 검색 지원

4. 게시물 상세: 
- 작성자 정보
- 게시글 내용
- 사진 첨부 기능
- 좋아요 버튼 (좋아요 취소 기능 제공 : 토글방식)
- 북마크 기능 (북마크 취소 기능 제공 : 토글방식)
- 댓글 기능(댓글 작성, 대댓글 작성, 좋아요 기능 (좋아요 취소 기능 제공 : 토글방식), 신고 기능)
- 게시물 및 댓글에 각각 적용 가능
- 좋아요 수 실시간 반영
- 사용자가 이미 좋아요한 경우 상태 표시 (활성/비활성 구분)
- 게시물 등록 버튼

4-1 글 작성 페이지 기능 
- 게시글 제목 입력
- 게시글 내용 작성 (텍스트 에디터)
- 이미지 업로드 기능 (버튼 누르면 랜덤 이미지 api사용해서 버튼 클릭하면 랜덤한 이미지가 추가되게)
- 카테고리 선택 (자취꿀팁 / 요리 / 공동구매 / 냉장고 나눔 / 동네정보 등)
- 지역 자동 태그 또는 선택 기능
- 공개/비공개 설정 기능
- 공동구매 / 나눔 게시물 선택 옵션 (상태 시스템 표시 : 모집중, 진행중, 마감완료)
- 작성 취소 / 임시저장 기능

5. 마이페이지 기능
- 사용자 프로필 정보 확인 및 수정 (닉네임, 프로필 이미지, 관심 지역)
- 내가 작성한 게시물 목록 확인 기능
- 내가 작성한 댓글 목록 확인 기능
- 북마크한 게시물 저장 목록 관리
- 활동 내역 확인 (좋아요, 댓글, 게시글 작성 기록)
- 지역 인증 상태 및 배지 확인

5-1 추가 기능 (운영/서비스)

== 커뮤니티 사이트 DB 구조 정리 ==

사용자 정보 (users 테이블) 
- 회원가입 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가) index id
  * 이름
  * 닉네임
  * 이메일  (로그인할 때 사용)
  * 비밀번호
  * 전화번호 
  * 가입일 (언제 가입했는지)

게시물 정보 (posts 테이블)
- 게시물 목록/상세 화면에서 필요한 정보들:
  * 게시물 번호: (자동으로 1, 2, 3... 증가) index id
  * 제목
  * 내용
  * 작성자: (어떤 사용자가 썼는지 - users 테이블과 연결)
  * 작성일
  * 수정일

댓글 정보 (comments 테이블)
- 댓글 화면에서 필요한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가) index id
  * 댓글 내용 
  * 작성자: (어떤 사용자가 썼는지 - users 테이블과 연결)
  * 게시물: (어떤 게시물에 달린 댓글인지 - posts 테이블과 연결)
  * 작성일

테이블 연결 관계
- 사용자 → 게시물: 한 사용자가 여러 게시물 작성 가능
- 사용자 → 댓글: 한 사용자가 여러 댓글 작성 가능
- 게시물 → 댓글: 한 게시물에 여러 댓글 작성 가능
