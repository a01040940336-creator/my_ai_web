포트폴리오 Contact 섹션 개발을 프로젝트 생성 규칙에 따라 진행해줘.

Supabase MCP를 사용해서 방명록 데이터베이스를 구축하고,
모든 개발이 완료되면 GitHub Pages로 최종 배포까지 진행해줘.
GitHub Actions 워크플로우를 사용해서 자동 배포되도록 설정하고, 배포 완료 후 접속 가능한 URL을 안내해줘.

⚠️ 중요: Contact 섹션은 Home 탭의 Contact 섹션을 개발하는 것입니다.
새로운 탭을 개발하면 안됩니다.

== Contact 섹션 UI 컨셉 및 느낌 ==

전체적인 느낌 (하나 선택)
☑ 모던하고 세련된 느낌
→ 여백 많고, 타이포그래피 중심 / 장식 최소화 / 고급스러운 미니멀

이메일/연락처 표시 방법
☑ 카드 안에 정리해서 깔끔하게
→ 한 줄 아이콘보다 “정보 블록화”가 더 럭셔리 느낌 잘 남

예:
Email
Phone
Location

각각 얇은 라인 + 작은 라벨 텍스트

SNS 링크 스타일
☑ 작은 아이콘들 한 줄로 정렬
→ hover 시 색상만 살짝 변화 (black → gray / gold 포인트 등)

색상 분위기
☑ 검정/흰색/회색 (심플한 무채색)
→ 필요하면 포인트 컬러 1개만 추가 (오프화이트 골드 or 차콜 블루 느낌)

배치 방법
☑ 세로로 차례대로 나열
→ 구조 추천:

상단: “Contact” 타이틀 (큰 타이포)
중간: 연락처 카드
하단: SNS 아이콘 + 방명록

특별히 원하는 스타일
- 참고 무드: Chanel / Dazed Korea editorial layout (여백 중심, 패션 매거진 느낌)
- 꼭 포함: 이메일 / 연락처/SNS 링크/방명록 (visitor message / guest form)

피하고 싶은 스타일:
- 컬러풀한 UI
- 캐릭터/아이콘 과다
- 앱 느낌의 버튼형 UI
- 과한 그림자/3D 효과

== 방명록에서 받을 정보 논의 ==

기본 필수 정보 (꼭 필요한 것들)
[ ] 방명록 ID (자동으로 생성되는 번호)
☑ 작성자 이름
☑ 방명록 내용/메시지
☑ 작성한 날짜/시간 (자동저장)

연락처 관련 (선택사항)
☑ 이메일 주소 (비공개로 저장)
☑ SNS 계정 (인스타, 트위터 등)

추가 정보 (있으면 좋은 것들)
☑ 소속/직업 (회사명일경우 직책, 학교일 경우 학교명 등)
[ ] 거주 지역 (시/도 정도만)
[ ] 나이대 (20대, 30대 등)
[ ] 어떻게 알게 되었는지

재미있는 정보 (개성 있게)
[ ] 좋아하는 색깔
☑ 한마디 키워드
[ ] 이모지 선택
[ ] 별점 평가

최종 결정된 정보 목록
1. 방명록 ID (자동 생성)
2. 작성자 이름
3. 방명록 내용/메시지
4. 작성 날짜/시간 (자동 저장)
5. 이메일 주소 (선택)
6. SNS 계정 (선택)
7. 한마디 키워드 (선택)
